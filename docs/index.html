Hệ Thống Quản Lý Đặc Tả Dự Án Parking

Tài liệu hướng dẫn cách thiết lập cấu trúc tính năng và quản trị nội dung đặc tả kỹ thuật cho hệ thống Tòa nhà Bãi đỗ xe thông minh (Parking Building Management System).

Giới thiệu chung về công cụ

Ứng dụng của bạn là một Feature Tree System được viết bằng công nghệ React, TypeScript và Tailwind CSS. Dữ liệu thiết kế được lưu trực tiếp trên trình duyệt bằng cơ sở dữ liệu IndexedDB local giúp toàn quyền bảo mật dữ liệu của bạn.

Hệ thống mẫu hiện tại mô phỏng dự án Parking Building Management System với hai đầu Backend dịch vụ: .NET Core API dành cho các nghiệp vụ ghi giao dịch chính, và Spring Boot Support API dành cho các báo cáo, tác vụ hỗ trợ.

Các phân hệ cốt lõi trong hệ thống Parking

Hệ thống mẫu đã được nạp sẵn sơ đồ nghiệp vụ chuẩn hóa cho bãi đỗ xe thông minh bao gồm các phân hệ sau:

Tên phân hệ Tính năng chi tiết tiêu biểu Vai trò hoạt động
1. Authentication Đăng nhập & Phiên làm việc, Đăng ký Tài xế khách Xác thực người dùng và phân quyền hệ thống.
2. User & Driver Quản lý tài khoản nội bộ, Hồ sơ xe, Đăng ký Vé tháng Quản trị phân quyền Admin/Manager/Staff và hồ sơ phương tiện.
3. Vehicle Config Cấu hình loại xe Thiết lập biểu phí dựa trên các loại phương tiện khác nhau.
4. Parking Structure Quản lý Tầng, Khu vực đỗ, Quản lý Vị trí (Slot) Cấu hình sơ đồ mặt bằng vật lý của bãi đỗ xe.
5. Entry & Exit Quét thẻ RFID tự động, Camera nhận diện biển số (ANPR) Tự động hóa luồng xe ra vào tại các barie an ninh.
6. Billing & Pricing Biểu giá (Tariff Matrix), Thanh toán cổng PayOS Tính toán giá vé và tích hợp thanh toán trực tuyến/quét QR.
7. Security & Admin Báo mất thẻ, Danh sách đen, Dashboard báo cáo Giám sát rủi ro mất cắp và kết xuất dữ liệu ca trực.

Quy trình điền đặc tả tính năng

Mỗi tính năng chi tiết (Leaf Feature) cần được mô tả đầy đủ qua 5 Tab nội dung chính để lập trình viên và Trợ lý AI có thể hiểu rõ:

1
Tab 1: Overview

Điền mục tiêu cốt lõi của tính năng, xác định rõ phạm vi (In Scope) và các hạng mục loại trừ (Out of Scope) cùng phân quyền đối tượng.

2
Tab 2: DB & Validation

Khai báo bảng dữ liệu SQL của PostgreSQL cần dùng và thiết lập các luật kiểm tra dữ liệu đầu vào kèm thông báo lỗi tương ứng.

3
Tab 3: Security & Logs

Định nghĩa chi tiết phân quyền API và các sự kiện ghi nhật ký nghiệp vụ quan trọng phục vụ mục đích kiểm toán (Audit Trail).

4
Tab 4: API & UI Contracts

Thiết lập URL API, JSON Request/Response mẫu. Đồng thời mô tả 4 trạng thái giao diện UI: Loading, Empty, Error và Success.

5
Tab 5: Tests & DoD

Viết kịch bản kiểm thử (Test Cases) chi tiết với các bước thực hiện và kết quả mong muốn cùng checklist tiêu chuẩn hoàn thành.

Mẫu giao việc cho đội ngũ phát triển

Sao chép mẫu dưới đây để gửi cho các thành viên trong dự án để họ hiểu rõ trách nhiệm và cách thức cộng tác thiết kế:

TASK_DELEGATION_TEMPLATE.md
### 📋 MẪU GIAO VIỆC: CẬP NHẬT ĐẶC TẢ TÍNH NĂNG HỆ THỐNG PARKING

> **Dự án:** Parking Building Management System (Hệ thống Quản lý Bãi đỗ xe)
> **Nhiệm vụ:** Hoàn thiện sơ đồ cây tính năng và cập nhật chi tiết đặc tả thiết kế hệ thống.

Chào các bạn,

Để chuẩn bị cho giai đoạn phát triển và kiểm thử hệ thống Parking, chúng ta cần hoàn thiện tài liệu đặc tả chi tiết cho từng tính năng trên công cụ Quản lý Đặc tả (Feature Tree).

Vui lòng đọc kỹ hướng dẫn dưới đây và thực hiện:

#### 1. Phân Chia Công Việc Theo Tính Năng (Feature-based Assignment):
* **Đảm nhận trọn gói tính năng:** Mỗi thành viên sẽ được phân công phụ trách cụ thể từng tính năng (Feature). Thành viên phụ trách tính năng nào sẽ chịu trách nhiệm hoàn thiện **toàn bộ các tab đặc tả (từ Tab 1 đến Tab 5)** của tính năng đó.
* **Mục tiêu:** Đảm bảo tính nhất quán cao nhất trong tư duy thiết kế — người thiết kế nghiệp vụ sẽ tự định nghĩa cơ sở dữ liệu, quy tắc bảo mật, API, giao diện UI và kịch bản kiểm thử tương ứng cho tính năng đó.

#### 2. Sử Dụng Trợ Lý AI Để Tự Động Hóa & Cập Nhật File Seed (Khuyên dùng):
* **Không cần điền thủ công trên giao diện:** Bạn không cần phải mở giao diện web lên và tự tay gõ từng dòng lệnh SQL, viết các JSON API Contract hay thiết kế từng bước Test Cases.
* **Quy trình thực hiện (Cập nhật trực tiếp vào mã nguồn):**
  1. Mở khung chat với Trợ lý AI (như Antigravity hoặc Gemini) trong môi trường phát triển của bạn.
  2. Gửi yêu cầu đặc tả bằng ngôn ngữ tự nhiên hoặc tài liệu tóm tắt cho AI (ví dụ: "Thêm 3 quy tắc validation và thiết lập API contract cho tính năng X").
  3. Yêu cầu AI **chỉnh sửa trực tiếp thông tin này vào file mã nguồn seed `src/seed/parkingBuildingSeed.ts`**.
  4. Sau khi AI thông báo đã chỉnh sửa thành công, bạn mở giao diện web của ứng dụng lên và nhấn nút **Nạp lại mẫu (Load Sample Seed)** để cập nhật hiển thị ngay lập tức.
  5. Thực hiện lệnh `git commit` và `git push` file `parkingBuildingSeed.ts` đã được cập nhật lên GitHub để đồng bộ cho toàn bộ đội ngũ.

#### 3. Nguyên Tắc Điền Thông Tin Cần Tuân Thủ:
* **Không bỏ trống các trường quan trọng:** Một tính năng được coi là "Ready for Dev" khi và chỉ khi có đủ: API Contract, Luật Validation dữ liệu, Quyền truy cập (Permissions), và ít nhất 2 Test Cases đi kèm.
* **Định dạng dữ liệu:**
  * API Contract phải viết dưới dạng mã JSON mẫu rõ ràng.
  * Câu lệnh SQL phải viết đúng cú pháp PostgreSQL để có thể chạy thử trực tiếp.
  * Quy định đặt tên Log Event phải viết chữ HOA, phân cách bằng dấu gạch dưới (ví dụ: `PAYMENT_COMPLETED`).
* **Đồng bộ hóa ngôn ngữ:** Ghi đặc tả bằng tiếng Việt rõ ràng, dễ hiểu hoặc tiếng Anh kỹ thuật đồng nhất.

*Sau khi hoàn thành cập nhật toàn bộ đặc tả cho tính năng được giao bằng cách chạy lại Seed, hãy chuyển trạng thái (Status) của tính năng đó từ `Draft` hoặc `In Progress` sang `Ready` để bàn giao cho đội ngũ phát triển.*
